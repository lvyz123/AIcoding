### 一、 运行可行性评估：无法在 12G 内存下顺利跑完

日志里有几个极其致命的“破案”数据：

1. **极其恐怖的版图密度：**
   `pattern 元素 1336570`。在一个 550um x 550um 的区块里塞了 133 万个图元，这是极其致密的先进工艺逻辑区（或者被打碎的复杂多边形）。这导致你的 Geometry-driven 策略在试图抓取 Residual（残余图形）时，**实际上退化成了全铺网格**。去重后的 seed 高达 `426838` 个，基本填满了所有的物理网格。
2. **候选对象引爆内存：**
   `candidate 生成完成: 1402890 个 ... 内存 RSS[rss_candidate_generation_mb] = 11081.371 MB`。
   在生成了 140 万个 `CandidateClip` 对象后，进程直接吃掉了 11.08 GB 的物理内存。对于 12GB 的电脑来说，留给操作系统和接下来极其耗费内存的 Coverage 计算的余量已经**完全没有了**。
3. **陷入死循环的“超级大桶”：**
   `coverage 轻量 bundle: bundle=1, 最大 group=914535, 总 group=914535`。
   因为所有的 Clip 尺寸都是强制统一的（比如 135x135 像素），所以多达 91.4 万个严格去重后的几何图形，被全部倒进了同一个 Shape Bucket (`bundle=1`) 里！
   接下来算法要在近 100 万个图形之间寻找近邻（这可是 $O(N^2)$ 级别的计算规模）。此时由于物理内存耗尽，操作系统开始疯狂地将内存数据写进硬盘虚拟内存（Swap Thrashing），导致原本几秒钟的 CPU 计算被拖延成几小时的硬盘读写，这就是为什么“下班了还没跑完”。

### 二、 下一步性能优化方案（突破 12G 内存封锁）

目前的瓶颈已经不是算法不够聪明，而是 **“海量 Python 对象的内存开销”** 和 **“超级大桶内的 $O(N^2)$ 比较”**。接下来的优化必须像做 C++ 引擎一样去压榨 Python：

#### 1. 彻底消灭 Python 对象，走向“结构体数组 (SoA)”
11GB 的内存峰值绝大多数是 Python 对象的 Overhead 造成的（一个 `CandidateClip` 对象加上它内部的字典、列表，开销可能大几百字节，而核心数据其实极少）。
* **方案：** 在 `_generate_candidates_for_cluster` 阶段，**不再生成 140 万个 `CandidateClip` 对象**。而是直接申请几个巨大的 NumPy 数组（比如 `shift_x_array`, `shift_y_array`, `origin_cluster_id_array`）。将所有特征压入 NumPy 矩阵，只在最后一步 Set Cover 选出胜者时，才将那几千个胜出者实例化为 Python 对象。这能把 11GB 的内存瞬间打到 1GB 以下。

#### 2. 粉碎超级大桶 (Mega Bucket)
把 91 万个图形放进一个桶里是绝对行不通的。虽然你代码里有 `_evaluate_bucketed_bundle_coverage`（按 Fill Bin 切分），但对于 133 万图元的版图，很可能绝大多数图形的填充率（Fill Ratio）都极其相似，导致切分失效。
* **方案：** 引入更强力的 **“复合分桶 (Composite Bucketing)”**。在 Coverage 阶段，不仅按 shape 和 fill-ratio 分桶，还要按 `_cheap_bitmap_descriptor` 里的 `peak_grid`（能量最高所在的象限，0-99）进行强制硬切分。把 91 万的桶切成几百个只有几千元素的小桶，HNSW 检索和矩阵运算才能在几秒内跑完。

#### 3. 动态遏制 Residual Seed 爆炸
133 万的图元让 Residual 机制彻底失控。
* **方案：** 在生成 Residual Seed 前，评估当前 Grid Cell 内的图元密度。如果某个 0.675um 的网格内已经包含了极其密集的图形（比如超过 50 个 polygon），直接跳过 Residual 采样，或者强行将该区域的 `grid_step_ratio` 放大到 0.8。对于这种极度密集的“毛刺”区，盲目高密度采样带来的召回率提升，远抵不过它造成的系统崩溃风险。

这套优化方案能彻底打掉当前 11GB 的内存巨兽。在你看来，这 133 万个元素的版图，主要是打散的走线（Routing），还是包含了极其复杂的基础单元（Standard Cells）内部结构？（了解图形特征有助于我们设计更具针对性的抛弃策略）。
